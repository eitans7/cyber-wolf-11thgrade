Game Model

Assumptions

  1. There are 8 users initially.
  2. One user is randomly chosen as the wolf.
  3. During the day, users can chat to discuss and identify the wolf.
  4. Each user (including the wolf) randomly votes for another user during the day.
  5. During the night, the wolf randomly kills one user.

States

  • S0: Registration
  • S1: Game Start
  • S2: Day
  • S3: Voting
  • S4: Night
  • S5: Wolf Killed
  • S6: Wolf Kills
  • S7: Wolf Wins
  • S8: Game Over

Events

  • E0: All users registered
  • E1: Wolf selected
  • E2: Day timer ends
  • E3: Majority vote for wolf
  • E4: No majority vote
  • E5: Night timer ends
  • E6: Wolf kills all but one
  • E7: Wolf identified and killed
  • E8: Wolf wins

Graphical Representation

Transitions and Probabilities

  1. (S0, E0) → S1 (All users registered)
  2. (S1, E1) → S2 (Wolf selected)
  3. (S2, E2) → S3 (Day timer ends)

During Voting (S3):

  • Probability of each user voting for any other user: 1/7 (assuming 7 other users are alive)
  1. (S3, E3) → S5 (Majority vote for wolf)
  2. (S3, E4) → S4 (No majority vote)

During Night (S4):

  • Probability of the wolf killing any other user: 1/7 (assuming 7 other users are alive)
  1. (S4, E5) → S2 (Night timer ends)
  2. (S4, E6) → S6 (Wolf kills all but one)
  3. (S6, E5) → S2 (Night timer ends)
  4. (S6, E7) → S7 (Wolf wins)
  1. (S5, E7) → S8 (Wolf identified and killed)
  2. (S7, E8) → S8 (Wolf wins)

Mathematical Representation

State Transition Matrix (for Voting and Killing)

Let V be the voting matrix and K be the killing matrix.

Voting Matrix V


    Vij = {
        1/7  if user i votes for user j
        0    if i = j (a user cannot vote for themselves)
    }

Killing Matrix K


    Kij = {
        1/7  if wolf i kills user j
        0    if i = j (wolf cannot kill themselves)
    }

Simulation Algorithm

  1. Initialization:
    • Set initial state S0
    • Register 8 users
    • Randomly select one user as the wolf
  2. Game Start:
    • Transition to S1
  3. Day:
    • Transition to S2
    • Allow users to chat
    • End day after timer and transition to S3
  4. Voting:
    • Each user randomly votes
    • Check for majority vote
      • If wolf is voted out, transition to S5
      • Else, transition to S4
  5. Night:
    • Wolf randomly kills one user
    • If only one user remains, transition to S6
    • Else, transition to S2
  6. End Game:
    • Transition to S7 from S5 or S6

Conclusion

This game model illustrates the transitions and probabilities within a multiplayer game where one player is the wolf. It uses state transitions and probabilities to simulate game progression, showcasing the key mechanics and decision points.
